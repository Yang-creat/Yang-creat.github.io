11. 测试代码

11.1 测试函数

对于所编写的函数, 我们需要测试其健壮性, 即对各种情况都能够处理, 不至于在某种情况下使得程序崩溃. python中通过特定的结构来测试函数的健壮性.

一. 单元测试和测试用例

使用模块unittest进行代码测试. 单元测试用于核实函数某个方面没有问题; 测试用例是一组单元测试, 这些单元测试一起测试函数在各种情况下行为都符合要求.

二. 可通过的测试
  1. 导入unittest模块以及测试函数
  2. 创建测试类, 命名包括Test字样, 并继承unittest.TestCase类
  3. 定义测试方法, 命名必须以test开头, 系统会自动运行
  4. 方法内部调用了函数以及使用了测试实参, 并通过断言将结果与标准结果比对, 若函数结果与标准结果相同, 则会返回ok, 否则返回failed.
  5. 通过unittest.main()自动执行.
    三. 测试若未通过怎么办

    说明函数结果有错误, 需要在此实参下函数不能得到正确结果, 需要修改函数.

四. 添加新测试

可以通过添加另一个方法用来测试其他的实例是否正确.

11.2 测试类

一. 各种断言方法

除了上述的assertEqual断言方法还有其他的断言方法.

  1. assertEqual(a, b), 核实a == b
  2. assertNotEqual(a, b, 核实a != b
  3. assertTrue(x), 核实x为True
  4. assertFalse(x), 核实x为False
  5. assertIn(item, list), 核实item在list中
  6. assertNotIn(item, list), 核实item不在list中
    二. 方法setUp()
    在unittest.TestCase类中包含了方法setUp(), 使得我们只需要创建这些对象一次, 在每个测试方法中都可以使用他们

在setUp中初始化了三个元素, 在后面的测试方法中可以直接使用.
